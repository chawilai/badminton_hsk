1. ภาพรวมระบบ

ระบบ annotation ใช้ 2 เทคโนโลยีร่วมกัน:

เทคโนโลยีหน้าที่ทำไมถึงเลือก
Sharp อ่าน/เขียนรูปภาพ, วาง SVG overlay ทับ เร็วมาก (ใช้ libvips), รองรับ PNG/JPEG/WebP
SVG วาดกรอบ, ลูกศร, ข้อความ, วงกลม Vector = คมชัดทุกขนาด, เขียน code ได้ง่าย

Flow: รูปต้นฉบับ (.png) → Sharp อ่านรูป → สร้าง SVG overlay ตามพิกัด → Sharp วาง SVG ทับ → บันทึกเป็นรูปใหม่

2. ติดตั้ง

# ติดตั้ง sharp
npm install sharp

# ตรวจสอบว่าใช้ได้
node -e "const sharp = require('sharp'); console.log('sharp version:', sharp.versions.sharp);"
Sharp ต้องการ Node.js 18+ และจะดาวน์โหลด native binary (libvips) ให้อัตโนมัติ ใช้ได้ทั้ง macOS, Linux, Windows
ถ้าโปรเจคใช้ "type": "module" ใน package.json ให้ตั้งชื่อไฟล์เป็น .cjs แทน .js

5. เครื่องมือทั้ง 6 แบบ

5.1 กรอบสี่เหลี่ยม (rect)

ใช้ครอบส่วนของ UI ที่ต้องการอธิบาย

{ type: 'rect', x: 38, y: 520, w: 314, h: 52, color: '#22c55e', strokeWidth: 3 }

// กรอบเส้นประ
{ type: 'rect', x: 30, y: 453, w: 335, h: 340, color: '#3b82f6', dashed: true }
ParameterTypeDefaultคำอธิบาย
xnumber—ระยะจากขอบซ้ายถึงมุมซ้ายบนของกรอบ (px)
ynumber—ระยะจากขอบบนถึงมุมซ้ายบนของกรอบ (px)
wnumber—ความกว้างของกรอบ (px)
hnumber—ความสูงของกรอบ (px)
colorstring#ef4444สีของเส้นกรอบ (hex)
strokeWidthnumber3ความหนาของเส้น (px)
radiusnumber8ความโค้งของมุม (px, 0 = มุมเหลี่ยม)
dashedbooleanfalsetrue = เส้นประ
ใช้ dashed: true สำหรับกรอบที่เป็นข้อมูลเสริม ไม่ใช่จุดเด่นหลัก

5.2 ลูกศร (arrow)

ใช้ชี้จากข้อความไปยังจุดที่ต้องการ

{ type: 'arrow', x1: 215, y1: 80, x2: 270, y2: 95, color: '#ef4444', strokeWidth: 3 }
ParameterTypeDefaultคำอธิบาย
x1, y1number—จุดเริ่มต้นของลูกศร (ด้านหาง)
x2, y2number—จุดปลายของลูกศร (ด้านหัว/ปลายแหลม)
colorstring#ef4444สีลูกศร (รองรับ: แดง, น้ำเงิน, เขียว, ส้ม)
strokeWidthnumber3ความหนาของเส้น
หัวลูกศรใช้ SVG marker — สีที่รองรับมี 4 สี: #ef4444 (แดง), #3b82f6 (น้ำเงิน), #22c55e (เขียว), #f97316 (ส้ม) ถ้าใช้สีอื่น หัวลูกศรจะไม่แสดง

5.3 ป้ายข้อความ (label)

ข้อความพร้อมพื้นหลังสีขาว ใช้อธิบายสิ่งที่ชี้ไป

{ type: 'label', x: 80, y: 512, text: 'กดเข้าสู่ระบบผ่าน LINE', color: '#22c55e', fontSize: 13 }
ParameterTypeDefaultคำอธิบาย
xnumber—ตำแหน่ง x ของข้อความ (ด้านซ้าย)
ynumber—ตำแหน่ง y ของ baseline ข้อความ (ไม่ใช่ขอบบน!)
textstring—ข้อความที่แสดง
colorstring#ef4444สีข้อความ + สีกรอบ
bgstringwhiteสีพื้นหลัง
fontSizenumber14ขนาดตัวอักษร (px)
ตำแหน่ง y คือ baseline ของข้อความ (ด้านล่างตัวอักษร) ไม่ใช่ขอบบน! กรอบพื้นหลังจะวาดขึ้นไปจาก y โดยอัตโนมัติ ดังนั้น y ควรอยู่ ต่ำกว่า ตำแหน่งที่ต้องการเล็กน้อย

5.4 ตัวเลขในวงกลม (number)

ใช้แสดงลำดับขั้นตอน เช่น (1) (2) (3)

{ type: 'number', x: 18, y: 546, num: 1, color: '#22c55e', size: 28 }
ParameterTypeDefaultคำอธิบาย
xnumber—ตำแหน่ง x จุดกลาง วงกลม
ynumber—ตำแหน่ง y จุดกลาง วงกลม
numnumber—ตัวเลขที่แสดง (1-99)
colorstring#ef4444สีพื้นหลังวงกลม (ตัวเลขสีขาวเสมอ)
sizenumber28เส้นผ่าศูนย์กลางวงกลม (px)
วาง number ด้านซ้ายของ rect เพื่อบอกลำดับ เช่น (1) ชื่อปาร์ตี้ (2) วันเล่น (3) จำนวนคน

5.5 วงกลมเปล่า (circle)

ใช้วนรอบปุ่มเล็ก ๆ เช่น ปุ่ม + หรือไอคอน

{ type: 'circle', x: 350, y: 93, r: 16, color: '#22c55e', strokeWidth: 3 }
ParameterTypeDefaultคำอธิบาย
xnumber—ตำแหน่ง x จุดกลาง
ynumber—ตำแหน่ง y จุดกลาง
rnumber20รัศมี (px)
colorstring#ef4444สีเส้นวงกลม
strokeWidthnumber3ความหนาของเส้น

5.6 การใช้ร่วมกัน (Combo Pattern)

ปกติจะใช้หลายเครื่องมือร่วมกัน:

// Pattern: ตัวเลข + กรอบ + label
{ type: 'number', x: 16, y: 180, num: 1, color: '#ef4444', size: 22 },
{ type: 'rect', x: 30, y: 148, w: 345, h: 58, color: '#ef4444' },

// Pattern: label + ลูกศร + วงกลม (ชี้ปุ่ม)
{ type: 'label', x: 150, y: 74, text: 'สร้างแชทใหม่', color: '#22c55e' },
{ type: 'arrow', x1: 280, y1: 80, x2: 330, y2: 89, color: '#22c55e' },
{ type: 'circle', x: 350, y: 93, r: 16, color: '#22c55e' },

// Pattern: กรอบ + label ด้านล่าง
{ type: 'rect', x: 20, y: 142, w: 358, h: 148, color: '#22c55e' },
{ type: 'label', x: 30, y: 295, text: 'การ์ดปาร์ตี้', color: '#22c55e' },

6. สีที่ใช้ได้

สีHex Codeตัวอย่างใช้สำหรับ
แดง #ef4444 สีเริ่มต้น (default) จุดสำคัญ, ปุ่มหลัก, ลูกศรชี้
เขียว #22c55e ขั้นตอนหลัก, ปุ่ม LINE, สิ่งที่แนะนำ
น้ำเงิน #3b82f6 ข้อมูลเสริม, แท็บ, ตัวเลือกรอง
ส้ม #f97316 คำเตือน, สถานะ, toggle
ม่วง #8b5cf6 ข้อมูลเพิ่มเติม, หมายเหตุ
ลูกศร (arrow) รองรับเฉพาะ 4 สี: แดง, น้ำเงิน, เขียว, ส้ม เพราะหัวลูกศรใช้ SVG marker ที่กำหนดไว้ล่วงหน้า ส่วนเครื่องมืออื่นใช้สีอะไรก็ได้

7. วิธีหาพิกัด x, y

วิธีที่ 1: Preview (Mac) — ง่ายที่สุด

  1. เปิดรูปด้วย Preview
  2. เลือก Tools → Show Inspector (หรือ Cmd+I)
  3. เลื่อนเมาส์ไปบนรูป — ดูพิกัด x, y ที่ Inspector panel
  4. จด x, y ของมุมซ้ายบน + มุมขวาล่าง ของพื้นที่ที่ต้องการ
  5. คำนวณ: w = x_right - x_left, h = y_bottom - y_top

วิธีที่ 2: Playwright วัดอัตโนมัติ — แม่นที่สุด

const { chromium } = require('playwright');

(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage({ viewport: { width: 390, height: 844 } });

  await page.goto('http://your-app.test/login');

  // วัดตำแหน่งปุ่มจริง
  const btn = await page
    .locator('button')
    .filter({ hasText: /LINE/ })
    .first()
    .boundingBox();

  console.log(btn);
  // { x: 43, y: 525, width: 304, height: 42 }

  await browser.close();
})();
ใช้ Playwright วัดเมื่อต้องการความแม่นยำสูง หรือรูปมี element เยอะ ไม่ต้องเดาพิกัดเอง

วิธีที่ 3: เว็บ Online

  • pixspy.com — อัพโหลดรูป แล้วดูพิกัดเมาส์
  • image-map.net — วาดพื้นที่แล้วได้พิกัด

วิธีที่ 4: Browser DevTools

  1. เปิดรูปในเบราว์เซอร์ (ลาก drop ลง Chrome)
  2. กด F12 เปิด DevTools
  3. Tab Console → พิมพ์: 
    document.querySelector('img').addEventListener('click', e =>
  console.log(`x: ${e.offsetX}, y: ${e.offsetY}`)
);
  4. คลิกบนรูป — Console จะแสดงพิกัด

8. ตัวอย่างเต็ม — ทำ Annotation จากศูนย์

สมมติมีรูป my-screen.png (390x844) และต้องการ annotate 3 จุด:

ขั้นตอน 1 — วางรูปต้นฉบับ

cp my-screen.png tutorial/screenshots/my-screen.png

ขั้นตอน 2 — หาพิกัด

เปิดใน Preview แล้ววัด:

  • ปุ่ม A: มุมซ้ายบน (40, 200), ขนาด 300x45
  • ปุ่ม B: จุดกลาง (350, 90), รัศมี 16
  • พื้นที่ C: มุมซ้ายบน (20, 300), ขนาด 350x200

    { type: 'rect', x: 15, y: 296, w: 360, h: 208, color: '#3b82f6', dashed: true },
    { type: 'label', x: 25, y: 510, text: 'รายการข้อมูล', color: '#3b82f6' },

9. เทคนิคขั้นสูง

9.1 เว้นช่องระหว่างกรอบกับ element

ควรเว้น padding 4-6px รอบ element:

// ถ้า element อยู่ที่ x=43 y=525 w=304 h=42
// กรอบควรเป็น:
{ x: 38, y: 520, w: 314, h: 52 }
//   -5    -5   +10    +10  (padding 5px รอบด้าน)

9.2 วาง label ไม่ให้บัง content

  • วาง label ด้านบน กรอบ: y = rect.y - 8
  • วาง label ด้านล่าง กรอบ: y = rect.y + rect.h + 16
  • อย่าวาง label ทับ บนข้อความของ UI

9.3 วางตัวเลขให้ตรง

// number ใช้ x, y เป็นจุดกลาง
// ถ้ากรอบ rect อยู่ที่ y=520 h=52 → กึ่งกลาง = 520+26 = 546
{ type: 'number', x: 18, y: 546, num: 1 }
//                       ^ ตรงกลางแนวตั้งของกรอบ

9.4 ใช้สีเดียวกันสำหรับกลุ่มที่เกี่ยวข้อง

// ปุ่ม LINE → ทั้ง rect, number, label ใช้สีเขียวหมด
{ type: 'rect', ..., color: '#22c55e' },
{ type: 'number', ..., color: '#22c55e' },
{ type: 'label', ..., color: '#22c55e' },

// ปุ่ม Email → ใช้สีน้ำเงินหมด
{ type: 'rect', ..., color: '#3b82f6' },
{ type: 'number', ..., color: '#3b82f6' },
{ type: 'label', ..., color: '#3b82f6' },

9.5 Shadow effect

ทุกเครื่องมือมี drop shadow อัตโนมัติ (ผ่าน SVG filter) เพื่อให้เห็นชัดบนพื้นหลังสีเข้ม

10. API Reference — Sharp

คำสั่ง Sharp ที่ใช้ในโปรเจคนี้:

10.1 อ่านรูป

const sharp = require('sharp');

// อ่านรูปและดูข้อมูล
const image = sharp('input.png');
const metadata = await image.metadata();
console.log(metadata.width, metadata.height, metadata.format);
// 390 844 png

10.2 วาง SVG overlay ทับ

const svgBuffer = Buffer.from(`
  <svg xmlns="http://www.w3.org/2000/svg" width="390" height="844">
    <rect x="38" y="520" width="314" height="52"
      fill="none" stroke="#22c55e" stroke-width="3" rx="8"/>
  </svg>
`);

await sharp('input.png')
  .composite([{ input: svgBuffer, top: 0, left: 0 }])
  .png()
  .toFile('output.png');

10.3 Resize (ย่อ/ขยาย)

await sharp('input.png')
  .resize(200, 400)  // กว้าง 200, สูง 400
  .toFile('thumb.png');

// รักษาสัดส่วน — กำหนดแค่ width
await sharp('input.png')
  .resize({ width: 200 })
  .toFile('thumb.png');

10.4 Crop (ตัด)

await sharp('input.png')
  .extract({ left: 0, top: 764, width: 390, height: 80 })
  .toFile('bottom-nav.png');

10.5 แปลงฟอร์แมต

// PNG → JPEG (ลดขนาด)
await sharp('input.png').jpeg({ quality: 85 }).toFile('output.jpg');

// PNG → WebP (เล็กสุด)
await sharp('input.png').webp({ quality: 80 }).toFile('output.webp');

// JPEG → PNG
await sharp('input.jpg').png().toFile('output.png');

10.6 หมุน / พลิก

await sharp('input.png').rotate(90).toFile('rotated.png');
await sharp('input.png').flip().toFile('flipped.png');     // พลิกแนวตั้ง
await sharp('input.png').flop().toFile('flopped.png');   // พลิกแนวนอน

10.7 เบลอ / ปรับสี

await sharp('input.png').blur(5).toFile('blurred.png');
await sharp('input.png').grayscale().toFile('gray.png');
await sharp('input.png').tint({ r: 255, g: 0, b: 0 }).toFile('tinted.png');

10.8 ต่อรูปหลายรูป

// วางรูป B ทับบนรูป A ที่ตำแหน่ง (100, 200)
await sharp('background.png')
  .composite([
    { input: 'overlay1.png', top: 200, left: 100 },
    { input: 'overlay2.png', top: 400, left: 50 },
    { input: svgBuffer, top: 0, left: 0 },  // SVG ก็ได้
  ])
  .toFile('combined.png');

10.9 สร้างรูปเปล่า

// สร้างรูปเปล่าขนาด 400x300 สีแดง
await sharp({
  create: { width: 400, height: 300, channels: 4,
    background: { r: 239, g: 68, b: 68, alpha: 1 } }
}).png().toFile('red-box.png');

10.10 อ่านเป็น Buffer (ไม่บันทึกไฟล์)

const buffer = await sharp('input.png')
  .resize(100)
  .png()
  .toBuffer();

console.log(buffer.length); // ขนาด bytes

11. แก้ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้
กรอบเลื่อนไปด้านบน พิกัด y น้อยเกินไป เพิ่มค่า y ขึ้น (ลงมาในรูป)
กรอบเลื่อนไปด้านซ้าย พิกัด x น้อยเกินไป เพิ่มค่า x ขึ้น (ไปทางขวา)
label ทับข้อความ UI ตำแหน่ง y ของ label ตรงกับ UI เลื่อน label ขึ้น/ลง 20-30px
หัวลูกศรไม่แสดง ใช้สีที่ไม่มี marker กำหนดไว้ ใช้เฉพาะ 4 สี: แดง/น้ำเงิน/เขียว/ส้ม
ข้อความภาษาไทยไม่แสดง ระบบไม่มี Thai font SVG ใช้ font-family: sans-serif ควรมี Noto Sans Thai ในระบบ

const sharp = require('sharp');

const svg = Buffer.from(`
  <svg xmlns="http://www.w3.org/2000/svg" width="390" height="844">
    <rect x="38" y="520" width="314" height="52"
      fill="none" stroke="#22c55e" stroke-width="3" rx="8"/>

  </svg>

sharp('screenshot.png')
  .composite([{ input: svg, top: 0, left: 0 }])
  .toFile('annotated.png')
  .then(() => console.log('Done!'));

สูตร 2: Batch resize ทุกรูป

const sharp = require('sharp');
const fs = require('fs');
const path = require('path');

const dir = './screenshots';
const outDir = './thumbnails';
fs.mkdirSync(outDir, { recursive: true });

for (const file of fs.readdirSync(dir).filter(f => f.endsWith('.png'))) {
  await sharp(path.join(dir, file))
    .resize({ width: 200 })
    .toFile(path.join(outDir, file));
  console.log(`✅ ${file}`);
}
